# GymRivalz - Agent Guidelines & Operations Log

This document defines the rules, permissions, and history of actions performed by AI agents on the strava_plants codebase. All AI assistants interacting with this repository must read, understand, and strictly adhere to these instructions.

---

## 1. Agent Operational Rules

To streamline development and minimize friction, the following policies are established for all AI agent operations:

### 🚀 Git Integration
* **Automatic Pushes:** Every time a change is made to the codebase, it **must** be committed and pushed to the remote GitHub repository immediately.
* **No Push Permission Required:** You do **not** need to ask the user for permission before committing or pushing changes. Proceed with git operations autonomously.
* **Commit Messages:** Use clear, descriptive, and conventional commit messages (e.g., `feat: add badges collection`, `fix: correct user XP calculation`).

### 📁 File Creation
* **No File Creation Permission Required:** You do **not** need to ask the user for permission before creating new files or directories. Implement new components, utilities, and assets as needed.
